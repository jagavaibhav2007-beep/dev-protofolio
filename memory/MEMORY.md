# MEMORY - Premium Portfolio
> Stack: Planned Next.js App Router + TypeScript + Tailwind CSS + Framer Motion + Lucide React
> Updated: 2026-05-29

## Rules
1. Build the portfolio as responsive code, not by pasting component crop PNGs into the website.
2. Preserve the `template.png` layout, proportions, spacing, and visual hierarchy as the source of truth.
3. Use the user's avatar from `avartar/` for the final avatar placement.
4. Treat `components/images/` crops as reference assets only.
5. Keep implementation phases small enough for visual QA after each major section.

## Constraints
- The final website must be a 1:1 coded recreation of `template.png` with the user's avatar replacing the template avatar.
- `stack.md`, `style.md`, `components/`, and `plan.md` are the current build references.
- Do not use Redux.
- Do not introduce a backend unless the user explicitly asks for one.
- Do not import the component crop PNGs into the final portfolio UI.
